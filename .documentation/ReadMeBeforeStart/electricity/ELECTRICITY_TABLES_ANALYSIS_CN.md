## 理解消耗分类

消耗数据从**三个不同角度**进行组织：

### 1. 总体测量 (TL337/TL341)
- **内容**: 整个建筑或主要部分的总消耗
- **目的**: 跟踪整体能源使用趋势
- **时间跨度**: 594天（最长的历史数据）
- **关系**: TL337（累积）= 基准值 + sum(TL341 每小时增量)

### 2. 按电源相位分类 (TL342-345)
- **内容**: 同样的消耗按三相电源系统分解
- **目的**: 监控各相位之间的电力负载平衡
- **时间跨度**: 7天（2020年11月，新安装）
- **关系**: TL342（总计）= TL343（A相）+ TL344（B相）+ TL345（C相）
- **注意**: C相（16.5%）明显低于A相（41.2%）和B相（42.3%），表明负载不平衡

### 3. 按设备/用途分类 (TL208, TL209, TL211, TL212, TL213, TL4)
- **内容**: 按特定设备或功能区域分解的消耗
- **目的**: 了解哪些系统/设备消耗能源
- **分类方法**: 所有时间段使用相同的分类方式
- **可用数据**:
  - TL208: 供暖/制冷 (数据损坏, 2019-11)
  - TL209: 照明 (7天, 2019-11)
  - TL211: 设备/研发 (7天, 2019-11)
  - TL212: 电器 (7天, 2019-11)
  - TL213: Panel2A-1 (9个月, 2020-02 到 2020-11)
  - TL4: 通风系统 (7天, 2020-11)
- **注意**: 这是一个有效的分类维度；各表只是有不同时间段的数据

### 理想关系
```
TL341（总体）≈ TL342（相位总和）≈ TL213 + TL4 + 其他设备
```

但它们并不完全匹配，因为：
- 时间周期不同
- 测量点不同
- TL341可能只测量约20%的总消耗
- 设备分类不完整

### 多维度监控
这是一个**多维度监控系统**，从不同角度观察同一建筑的能源消耗：
- **时间维度**: 随时间的总体趋势
- **电力维度**: 供电系统角度（相位）
- **功能维度**: 设备/用途角度

## 理解发电分类

发电数据从**两个不同角度**进行组织：

### 1. 总体测量 (TL336/TL340)
- **内容**: 整个建筑的总发电量
- **目的**: 跟踪整体发电趋势和自给率
- **时间跨度**: 594天（与消耗数据对应）
- **关系**: TL336（累积）= 基准值 + sum(TL340 每小时增量)
- **模式**: 与消耗的TL337/TL341完全相同

### 2. 按发电源分类 (TL252, TL253)
- **内容**: 按太阳能板位置分解的发电功率
- **目的**: 监控不同位置太阳能板的性能
- **时间跨度**: 7天（2020年11月，新安装）
- **单位**: W（功率），不是Wh（能量）
- **关系**: TL252（车棚）+ TL253（屋顶）< TL340
  - 只测量了部分太阳能板
  - TL340可能包含其他发电源或更多太阳能板
  - 平均差异约73%，说明TL252/253只覆盖约27%的发电

### 3. 历史数据 (TL3)
- **内容**: 2019年11月的旧发电监控系统
- **时间**: 与TL208-212（消耗分类）同期
- **状态**: 已停用，**被TL340替代**（不是TL336）
- **说明**: TL3 测量总发电量，TL340 是同样功能的新系统

### 与消耗分类的对比

| 维度 | 消耗 (Consumption) | 发电 (Generation) |
|------|-------------------|-------------------|
| **总体测量** | TL337/TL341 ✓ | TL336/TL340 ✓ |
| **按相位分类** | TL342-345 (三相) ✓ | 无 ✗ |
| **按源/用途分类** | TL213, TL4 (设备) ✓ | TL252, TL253 (太阳能板) ✓ |
| **历史系统** | TL208-212 ✓ | TL3 ✓ |

**为什么发电没有相位分类？**
- 发电系统（如太阳能逆变器）通常会自动平衡三相输出
- 监控重点是发电源（哪个太阳能板），而不是电力分配
- 消耗需要相位监控是为了负载平衡，发电不需要

**系统替代历史：**
- **旧系统 (2019-11)**: TL3 (总发电) + TL208-212 (按设备消耗)
- **新系统 (2019-02 开始)**: TL340 (总发电) + TL341 (总消耗)
- **TL3 → TL340**: 两者都测量总发电量，TL340 是替代品
- **TL336**: TL340 的累积形式，不是 TL3 的替代品

### 理想关系
```
TL340（总发电）≈ TL252（车棚）+ TL253（屋顶）+ 其他太阳能板/发电源
```

但实际上：
```
TL340 ≈ 108 Wh
TL252 + TL253 ≈ 29 W (约27%)
差异 ≈ 79 Wh (约73%)
```

这说明：
- TL252/253 只监控了部分太阳能板
- 可能还有其他太阳能板没有被TL252/253监控
- 或者有其他发电源（但GBTAC主要是太阳能）

## 理解净能源分类

净能源数据的组织结构是**三个类别中最简单的**：

### 1. 总体测量 (TL338/TL339)
- **内容**: 建筑总净能源（发电 - 消耗）
- **目的**: 跟踪整体能源平衡和电网依赖度
- **时间跨度**: 594天（与消耗和发电数据对应）
- **关系**: TL338（累积）= 基准值 + sum(TL339 每小时增量)
- **模式**: 与 TL337/TL341（消耗）和 TL336/TL340（发电）完全相同
- **公式**: TL339 = TL340（发电）+ TL342（消耗）
  - 注意：消耗值是负数，所以实际上是 发电 - |消耗|

### 2. 备用测量 (TL335)
- **内容**: 备用净能源测量点
- **目的**: 不同的测量点或计算方法
- **时间跨度**: 7天（2020年11月，与 TL342-345 和 TL252-253 同期）
- **关系**: 与 TL339 相关但**不完全相同**（平均差异约 13.6%）
- **注意**: 
  - 数值与 TL339 在不同时间差异 10-30%
  - 变化方向相关性：85.7%（同向变化但幅度不同）
  - 可能测量建筑中不同位置的净能源
  - 或使用不同的计算方法

### 与其他类别的对比

| 维度 | 消耗 (Consumption) | 发电 (Generation) | 净能源 (Net Energy) |
|------|-------------------|-------------------|-------------------|
| **总体测量** | TL337/TL341 ✓ | TL336/TL340 ✓ | TL338/TL339 ✓ |
| **按相位分类** | TL342-345 (三相) ✓ | 无 ✗ | 无 ✗ |
| **按源/用途分类** | TL208-213/TL4 (设备) ✓ | TL252-253 (太阳能) ✓ | 无 ✗ |
| **备用/替代** | 无 ✗ | 无 ✗ | TL335 ✓ |
| **历史系统** | 无 (相同分类) | TL3 (被替代) ✓ | 无 ✗ |

**为什么净能源最简单？**
- 净能源是**计算结果**（发电 - 消耗）
- 不需要相位分解（已在消耗中）
- 不需要源分解（已在发电和消耗中）
- 只需要总体测量 + 备份验证

### 理想关系
```
TL339（净能源）= TL340（发电）+ TL342（消耗）
              = TL340 - |TL342|
```

实际情况：
- 当 TL339 为负：建筑需要电网供电（消耗 > 发电）
- 当 TL339 为正：建筑向电网输出（发电 > 消耗）
- GBTAC 通常为负（自给率仅约5%）
